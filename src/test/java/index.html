It is legal to hack this site, but please play nice!

We are programming students studying how to make websites secure. As part of our training, we allow this test site to be attacked, to learn what is working and what vulnerabilities remain. Hack away, but please do not deface the site with offensive material.

One hint for script kiddies!

If you're testing as part of our class, a basic username and password have been provided. For others, there is a user with a common name and a password that will be included in any test wordlist worth running. Regardless of how you gain access, this account has very limited priviliges. Beyond that, you are on your own.

If you are a hardened cyber-criminal -

You will probably get in if you try, because we are students and the security we are learning is still incomplete. You have nothing to gain though. All that's here is test data, which has no meaning except as a goal for us to defend.

Log in if you can!

Additional Information

This is not an intentionally-vulnerable site like The Bodget Store. We have a lot to learn, but we except for the one soft user account mentioned in the hint, we are making this site as stong as we can.

It will get stronger as the semester progresses, and we completely renew every few days, so if you manage to create a new user, the account will not last. Backdoors will vanish as well

You may be detected and possibly identified. The security skills we are learning include the ability to trace an attack, so that a criminal who attacked a production site could be prosicuted. However, hacking this test site is not a crime, so we won't make use of tracing information for any purpose beyond classroom exercises. However, if you want to keep your hacking games a secret, you have been warned!
